## Deal Properties Reference

This table describes all the information stored for each deal. Use this as a quick reference for understanding what data is captured and any rules that apply.

| Property | Data Type | What It Stores | Required? | Notes |
|----------|-----------|----------------|-----------|-------|
| **ID** | Number | Unique deal number | Auto-generated | Assigned automatically when deal is created. Cannot be changed. |
| **Status** | Text (max 20 chars) | Current pipeline stage | Yes (default: Inquiry) | One of: Inquiry, Discovery, Internal Review, Contracting, Won, Lost, Cancelled, or Declined.  |
| **Owner** | Reference → User | Team member responsible for this deal | No | Links to a user account. Can be reassigned at any time. |
| **Client** | Reference → Client | Company/organization for the event | Yes | Links to a client record. |
| **Primary Contact** | Reference → Person | Main person to communicate with | No | Links to a contact record. Contact must be linked to specified Client |
| **Max Budget** | Number | Client's maximum budget in USD | No | Stored in thousands (e.g., 50 = $50,000). Used for financial planning and approvals. |
| **Date Type** | Text (max 20 chars) | How dates are specified | No | One of: "Single Day" (one date), "Multi Day" (spans multiple days), or "Unconfirmed" (TBD). |
| **Primary Date** | Date | Main event date | No | Used when Date Type is "Single Day" or "Multi Day". Format: YYYY-MM-DD w/o time or timezone. Cannot be in the past. |
| **Is Date Flexible** | Yes/No | Whether client is willing to accomodate anternate dates for Primary Date  | No (default: No) | Toggle for Single Day or Multi Day events indicating client has some flexibility. Format: YYYY-MM-DD w/o time or timezone. |
| **Alternative Dates** | List of Dates | Backup date options | No | List of possible dates if primary doesn't work. Only for for Single Day or Multi Day events events with flexibility. Format: YYYY-MM-DD w/o time or timezone. |
| **Number of Days** | Number | How many days the event spans | No | Used with Primary Date for Multi Day events. Must be 2 or greater. |
| **Estimated Months** | List of Text | Months when event might occur | No | Used when Date Type is "Unconfirmed". Sequential months only (e.g., Jan-Mar, not Jan & May). Format: "YYYY-MM". |
| **Event Purpose** | Text (max 50 chars) | Why the event is happening | No | Common values: Social, Corporate, Brand. Helps categorize the opportunity. |
| **Event Format** | Text (max 50 chars) | Type of event | No | Examples: Dinner, Cocktail, Curated Trip, Gifting, Large Scale Activation. |
| **Services** | Text (max 50 chars) | What services are needed | No | Examples: Production, Marketing, Programming, RSVP. |
| **Guest Count** | Number | Expected number of attendees | No | Whole number. Important for venue sizing and staffing estimates. |
| **Locations** | Text Array | City and state where event will take place | No | List of US cities. Multiple locations allowed. Selected via Google Places search. Format: "New York, NY" |
| **Event Concept** | Long Text | Description of the event vision | No | Free-form text. Supports detailed descriptions. Used in search and AI summaries. |
| **Notes** | Long Text | General notes and comments | No | Internal notes about the deal. Not visible to clients. |
| **Created** | Date & Time | When deal was created | Auto-generated | Set automatically. Cannot be changed. |
| **Last Updated** | Date & Time | When deal was last modified | Auto-updated | Updates automatically whenever any field changes. |


## Client Properties Reference

Clients represent companies or organizations that may host events. Each deal is linked to a client record.

| Property | Data Type | What It Stores | Required? | Notes |
|----------|-----------|----------------|-----------|-------|
| **ID** | Number | Unique client number | Auto-generated | Assigned automatically when client is created. Cannot be changed. |
| **Name** | Text (max 255 chars) | Company or organization name | Yes | Must be provided. Used for matching inquiries to existing clients. Searchable. |
| **Industry** | Text (max 255 chars) | Business sector or category | No | Examples: Technology, Finance, Entertainment, Fashion. Helps with categorization and reporting. Searchable. |
| **Domain** | Text (max 255 chars) | Company website domain | No | Example: "acme.com". Used for identifying and deduplicating clients. |
| **About** | Long Text | Description of the company | No | Background information, history, or notable details about the organization. |
| **Notes** | Long Text | Internal notes about the client | No | Private notes for the team. Not visible to clients. |
| **Created** | Date & Time | When client record was created | Auto-generated | Set automatically. Cannot be changed. |

### Client Relationships

- **Deals**: A client can have multiple deals (one-to-many)
- **Contacts**: A client can have multiple contacts, and contacts can belong to multiple clients (many-to-many via Client Contacts table)


## Contact Properties Reference

Contacts represent individuals associated with clients. A contact can be linked to multiple clients.

| Property | Data Type | What It Stores | Required? | Notes |
|----------|-----------|----------------|-----------|-------|
| **ID** | Number | Unique contact number | Auto-generated | Assigned automatically when contact is created. Cannot be changed. |
| **First Name** | Text (max 255 chars) | Person's first name | Yes | Must be provided. Searchable. |
| **Last Name** | Text (max 255 chars) | Person's last name | No | Searchable. Combined with first name for display. |
| **Title** | Text (max 255 chars) | Job title or role | No | Examples: "CEO", "Event Manager", "Marketing Director". |
| **Email** | Text (max 255 chars) | Email address | No | Used for communication and matching incoming emails. Searchable. |
| **Phone** | Text (max 50 chars) | Phone number | No | Any format accepted (e.g., "(555) 123-4567" or "+1-555-123-4567"). |
| **LinkedIn Username** | Text (max 255 chars) | LinkedIn profile username | No | Just the username portion (e.g., "johndoe" not the full URL). Used to link to their profile. |
| **Created** | Date & Time | When contact record was created | Auto-generated | Set automatically. Cannot be changed. |

### Contact Relationships

- **Clients**: A contact can be associated with multiple clients (many-to-many via Client Contacts table)
- **Deals**: A contact can be the primary contact for multiple deals

## Client-Contact Association

The link between clients and contacts includes:

| Property | Data Type | What It Stores | Required? | Notes |
|----------|-----------|----------------|-----------|-------|
| **Client** | Reference → Client | Which client | Yes | The client this contact is associated with. |
| **Contact** | Reference → Contact | Which contact | Yes | The contact being associated. |
| **Active** | Boolean | Whether association is active | Yes (default: true) | Set to false when a contact leaves a company. |
| **Created** | Date & Time | When association was created | Auto-generated | Set automatically. |
